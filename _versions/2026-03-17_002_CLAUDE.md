# Reglas de Interacción para Claude Code (Subordinado a Crisalia QA)

Eres un desarrollador trabajando en el proyecto Crisalia. Tienes un supervisor de QA automatizado (Crisalia Dialectic Engine).

## Protocolo de Validación — Activación por Marca

El Tribunal Dialéctico de Crisalia NO se ejecuta automáticamente en cada tarea.

**Regla principal:**
- Si el prompt del usuario **NO termina en `q.`** → realiza la tarea, guarda los cambios y finaliza normalmente. No ejecutes el validador.
- Si el prompt del usuario **termina en `q.`** → realiza la tarea y, antes de finalizar, ejecuta obligatoriamente `python crisalia_bridge.py` y presenta el veredicto completo al usuario.

## Procedimiento cuando se activa el Tribunal (`q.`)

1. Guarda tus cambios localmente.
2. Ejecuta: `python -X utf8 crisalia_bridge.py`
3. Lee atentamente la salida de la consola.
   - Si el veredicto es "APROBADO": Tienes luz verde. Informa al usuario y finaliza.
   - Si el veredicto es "RECHAZADO" o "APROBADO_CON_OBSERVACIONES": Lee "Sugerencias" y "Riesgos", aplica las correcciones y vuelve a ejecutar el paso 2.
4. CIRCUIT BREAKER: Máximo 2 intentos de corrección. Si Crisalia rechaza 3 veces seguidas, detente y pedile intervención humana al usuario.
